Presentation layer providing HTTP request/response helpers, input parsing, and middleware for API and CLI applications. Package name: `tkPresentation`.

## Summary

- middleware/ — request processing middleware (logging, panic handling)
- honeypot/ — honeypot middleware with embed.FS payload loading (114 base64-encoded .bin files: 25 original + 89 expanded across 17 vulnerability categories)
- honeypotMiddleware.go — `*HoneypotMiddleware` struct with constructor `NewHoneypotMiddleware(settings, honeypotCmdRepo, honeypotQueryRepo, activityRecordCmdRepo)`, `MiddlewareFunc() echo.MiddlewareFunc`, `Start()`, `Stop()`; three-class path system (`HoneypotPathClassStaticVuln`, `HoneypotPathClassBandwidthExhaust`, `HoneypotPathClassAITrap`); `Execute()` calls use cases directly (`ReadHoneypotBanDecision`, `CreateHoneypotHit`) passing VO-typed params; `lookupActivePathClass` checks if request path is active; `dispatchByPathClass` routes to static payload, bandwidth exhaust streaming, or AI trap streaming; tier 2/3 delegates to `serveMixedResponse`; `recordHoneypotHit` creates activity record for each hit; `honeypotMaintenanceWatchdog` goroutine ticks at StatsInterval calling `runMaintenance`; no closure fields — repos stored as struct fields; `writeMu sync.Mutex` protects write path; `activePathClasses` map pre-computed in constructor via auto-ratio selection; constructor calls `buildHoneypotPayloadMap(activePathMap, extraRoutes)` which decodes only active static paths from base64 .bin files; decode failures logged via `slog.Error`, path skipped, replacement selected from dormant pool; under 300 LOC; zero else keywords; methods ordered callees-above-callers with constructor last
- honeypotPathMapping.go — `honeypotPayloadSpec` struct (urlPath, binFileName, mimeType); `honeypotPayloadSpecs` slice with 114 entries mapping URL paths to base64-encoded .bin filenames and MIME types; `findPayloadSpec(interceptPath)` looks up spec by URL path; categories: env variants, WordPress, Git/SVN, package managers, config files, backup/archive, info/debug, uploads/temp, admin/login, API/debug, web/crossdomain, webhooks, logs, cloud/storage, build artifacts, path traversal, other
- honeypotPathSelector.go — `//go:embed honeypot/payloads/*.bin` embed directive; `HoneypotPathClass` type + constants; candidate pools (10 bandwidth exhaust, 10 AI trap); `decodePayloadSpec(spec)` reads .bin from embed.FS and base64-decodes into `HoneypotPathMapping` with `UrlPath`/`MimeType` VOs; `buildHoneypotPayloadMap(activePathMap, extraRoutes)` decodes only active static paths, handles failures with `resolveDecodeFailures` replacement selection; `extractStaticPathKeys()` returns path strings from specs; `computeAutoRatio(activePathCount)` returns (static, bandwidth, aiTrap) counts with floor of 1 per class; `selectActivePaths` uses `math/rand` with configurable seed (0 = time-based); `shuffleAndTake` selects random subset from candidate pool
- honeypotAiTrapGenerator.go — `honeypotAiTrapGenerator` struct; `generateChunk(interceptPath, chunkIndex)` produces plausible content with embedded prompt injection patterns (ANALYSIS_TASK, COMPUTE_REQUEST, HASH_TASK, TRACE_TASK, DEEP_ANALYSIS); content type varies by path (JSON for docs/status, text/plain for metrics); injection hidden in JSON metadata fields
- honeypotStreamHandler.go — `streamBandwidthExhaust(echoContext)` streams garbage text/plain via Flusher with random cap between 5MB floor and MaxStreamSizeBytes; `streamAiTrap(echoContext, path)` streams AI trap content; `serveStreamFallback(echoContext)` returns 200 + valid JSON when Flusher unavailable; `computeStreamCap(maxSize, rng)` returns random int64 in [floor, max) range; streaming checks request context for client disconnect
- honeypotMixedResponse.go — mixed response helpers split from main middleware to respect 300 LOC cap; `serveMixedResponse` weighted random selector (40% LE redirect, 30% fake 503, 20% fake 502, 10% fake 429); `serveLawEnforcementRedirect` picks random URL from FBI/NSA/Interpol pool with rotating security-themed query strings and mixed 302/307; custom redirect URL (non-default) overrides pool with 100% redirect, no query string; `fake503Body`/`fake502Body` return nginx-style HTML; `fake429Body` returns valid JSON; `isMixedResponseStatusCode` helper for tests; `hasCustomRedirect` checks against default URL constant
- honeypotSettingsParser.go — `honeypotSettingsParser` struct with `Parse(settings, poolCeiling)` method that resolves env vars into typed VOs; sub-methods per env var: `parseBanDuration`, `parseRedirectUrl`, `parseMaxEntries`, `parseActivePathCount` (applies default 30 when zero, clamps settings values to poolCeiling), `parseMaxStreamSizeBytes`, `parseStatsInterval`, `parseAggressivenessMode`; all infra errors logged via `slog.Error`
- envsInspector.go — loads .env files, validates required environment variables, and auto-fills missing auto-fillable variables with generated secret keys
- paginationParser.go — parses pagination parameters from untrusted input maps into Pagination DTOs
- requestInputReader.go — reads and merges HTTP request input from path params, query params, headers, and body (JSON/form)
- requesterIpExtractor.go — struct with constructor NewRequesterIpExtractor() that pre-computes an ordered header chain (IP_EXTRACT_HEADER, default: X-Forwarded-For,X-Real-IP) and trusted CIDR blocks from tkInfra.TrustedCidrsReader(); Execute(*http.Request) returns (IpAddress, error), tries headers first with right-to-left trust walk (IsLocal/IsPrivate/IsLinkLocal/CidrBlock.Contains), supports Direct/RemoteAddr keywords, implicit RemoteAddr fallback; no echo dependency
- requiredParamsInspector.go — checks that all required parameters are present in an input map
- responseWrappers.go — API and CLI response formatting (ApiResponseWrapper, LiaisonCliResponseRenderer with syntax-highlighted JSON, SimpleCliResponseRenderer delegates to LiaisonCliResponseRenderer for simplified CLI usage accepting isSuccess bool and message string)
- stringSliceVoParser.go — generic parser converting raw input into typed value object slices
- timeParamsParser.go — parses optional time parameters from untrusted input into UnixTime pointers
- *_test.go — tests for each component
