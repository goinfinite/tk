# Middleware Context

## Purpose

Middleware functions process HTTP requests before they reach controllers. They handle cross-cutting concerns like logging, panic recovery, authentication, and database injection.

## Files

- `panicHandler.go`: Catch panics, log stack traces, return error responses
- `logHandler.go`: Configure logging levels via environment variables
- `logs/`: Logging utilities and helpers

## Critical Rules

- **Order matters**: Middleware execution order affects behavior
- **Early returns**: Middleware can short-circuit the request chain
- **Echo framework**: Middleware designed for Echo web framework
- **Reusable**: Middleware should be framework-agnostic when possible

## Panic Handler

### API Panic Handler

- Catches panics in HTTP request handlers
- Filters stack traces (excludes domain layers for clarity)
- Returns HTTP 500 with error message
- Masks sensitive info based on `TRUSTED_IPS` environment variable

### CLI Panic Handler

- Recovers from panics in CLI applications
- Logs error and exits gracefully
- Use with `defer CliPanicHandler()`

## Log Handler

Configures logging based on `LOG_LEVEL` environment variable:

- Supports slog and Zerolog
- Default level: Info
- Available levels: Debug, Info, Warn, Error

## Middleware Pattern (Echo)

```go
func MiddlewareName(next echo.HandlerFunc) echo.HandlerFunc {
    return func(c echo.Context) error {
        // Pre-processing

        // Call next handler
        err := next(c)

        // Post-processing

        return err
    }
}
```
