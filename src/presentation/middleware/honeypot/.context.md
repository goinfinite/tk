Deceptive response infrastructure for the honeypot middleware. Package name: `tkPresentationMiddlewareHoneypot`. Provides path pool management, response handlers (static payload, bandwidth exhaust stream, AI trap, mixed/redirect), and payload loading.

## Summary

- honeypotPathPool.go — enum of all possible honeypot paths organized by class (admin, config, backup, api, debug, etc.)
- honeypotPathSelector.go — selects a subset of active paths from the pool based on settings and random seed
- honeypotPathMapping.go — resolves incoming request URL paths to their honeypot path class
- honeypotSettingsParser.go — parses and validates raw settings input
- streamHandler.go — serves continuous bandwidth exhaust stream and AI trap stream responses
- aiTrapGenerator.go — generates AI-simulated administrative interface content
- mixedResponseHandler.go — serves mixed/redirect responses (e.g., HTTP 302 redirect to external targets)
- payloadLoader.go — loads static payload files from the embedded payloads/ directory
- payloads/ — embedded static payload files for payload-based responses
- testdata/ — test fixtures for handler tests
- *_test.go — tests for each component
