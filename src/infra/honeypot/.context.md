Repository implementations for honeypot hit persistence. Package name: `tkInfraHoneypot`. Uses the transient in-memory SQLite database; no persistent storage.

## Summary

- honeypotCmdRepo.go — command repository: Create (insert hit), DeleteExpired (TTL-based cleanup), EnforceMaxEntries (entry cap)
- honeypotQueryRepo.go — query repository: ReadBanDecision (hit count by IP), ReadStatsReport (aggregated statistics)
- *_test.go — tests for both repositories with in-memory database

## Constraints

- MUST use `TransientDatabaseService` — never a persistent database.
- MUST NOT depend on any env vars — configuration comes through DTOs and VOs.
