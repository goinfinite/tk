Repository implementations for honeypot hit tracking, cleanup, and stats aggregation. Implements the interfaces defined in `domain/repository/`. Package name: `tkInfraHoneypot`.

## Summary

- honeypotCmdRepo.go — IncrementHit (read-modify-write JSON in transient DB), CleanExpiredEntries (GORM delete by age), EnforceMaxEntries (GORM delete oldest over limit)
- honeypotQueryRepo.go — ReadHitRecord (read and parse JSON from transient DB), Count (delegate to transient DB), ReadReport (full stats aggregation with tier resolution)

## Constraints

- MUST satisfy `tkRepository.HoneypotCmdRepo` and `tkRepository.HoneypotQueryRepo` interfaces.
- MUST use `tkInfraDb.TransientDatabaseService` for database access.
