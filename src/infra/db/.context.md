# Database Context

## Purpose

Provides database connection management, services, and query-building utilities used by repository implementations.

## Files

- `trailDatabaseService.go`: SQLite trail database initialization and migration
- `paginationQueryBuilder.go`: Builds paginated GORM queries from pagination DTOs
- Tests for both services

## Critical Rules

- **GORM-based**: All database operations use GORM ORM
- **SQLite**: Trail database uses SQLite for activity records
- **Migration support**: Auto-migrate models on service initialization
- **Environment-driven**: Database paths configured via environment variables

## Trail Database Service

### Purpose

Manages the trail database (activity records) with GORM.

### Configuration

- `TRAIL_DATABASE_FILE_PATH`: Path to SQLite database file
- Default models: `ActivityRecord`, `ActivityRecordAffectedResource`
- Supports additional models via constructor parameter

### Usage Pattern

```go
trailDbSvc, err := NewTrailDatabaseService([]any{&CustomModel{}})
// Access GORM handler
trailDbSvc.Handler.Model(&ActivityRecord{}).Find(&records)
```

## Pagination Query Builder

### Purpose

Builds paginated GORM queries with sorting and limit/offset support.

### Features

- Page number or last-seen-ID pagination
- Configurable items per page
- Sort by field and direction
- Total count calculation

### Usage Pattern

```go
query := db.Model(&Model{})
pagination := tkDto.Pagination{PageNumber: 0, ItemsPerPage: 10}
paginatedQuery, responsePagination, err := PaginationQueryBuilder(query, pagination)
```
