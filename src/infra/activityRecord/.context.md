# Activity Record Infrastructure Context

## Purpose

Implements repository interfaces for activity record persistence, transforming between domain entities and database models.

## Files

- `activityRecordCmdRepo.go`: Create and Delete operations
- `activityRecordQueryRepo.go`: Read operations with pagination and filtering
- Tests for both repositories

## Critical Rules

- **Implement interfaces**: Must satisfy `tkRepository.ActivityRecordCmdRepo` and `ActivityRecordQueryRepo`
- **Use database models**: Transform DTOs to `tkInfraDbModel.ActivityRecord`
- **Return entities**: Transform models to `tkEntity.ActivityRecord` for read operations
- **Trail database**: All operations use `TrailDatabaseService`

## Transformation Flow

### Create/Delete (CmdRepo)

```
DTO → Database Model → GORM Create/Delete
```

### Read (QueryRepo)

```
DTO (filters) → GORM Query → Database Models → Entities → Response DTO
```

## Common Patterns

- JSON serialization for `RecordDetails` field
- Null handling for optional fields (`OperatorSri`, `OperatorIpAddress`)
- Associated resources loaded via GORM relationships
- Pagination via `PaginationQueryBuilder`
