# Repository Context

## Purpose

Repository interfaces define contracts for data access. The domain layer defines WHAT operations are needed; the infrastructure layer implements HOW they work.

## Critical Rules

- **Interfaces only**: This directory contains only interface definitions
- **Implementation in infra**: Actual implementations are in `src/infra/{entity}/`
- **Domain types**: All parameters and return values use domain types (entities, DTOs, value objects)
- **Separation of concerns**: Query repos (read) separate from Cmd repos (write)

## Repository Types

### Command Repository (CmdRepo)

Write operations: Create, Update, Delete

```go
type EntityCmdRepo interface {
    Create(createDto tkDto.CreateEntity) error
    Update(updateDto tkDto.UpdateEntity) error
    Delete(deleteDto tkDto.DeleteEntity) error
}
```

### Query Repository (QueryRepo)

Read operations with pagination support

```go
type EntityQueryRepo interface {
    Read(requestDto tkDto.ReadEntitiesRequest) (tkDto.ReadEntitiesResponse, error)
}
```

## Naming Conventions

- Interface: `{Entity}CmdRepo` or `{Entity}QueryRepo`
- Methods: Use clear action verbs (Create, Read, Update, Delete)
- Parameters: Accept DTOs, not individual fields
- Returns: Return error or (response DTO, error)

## Method Signatures

- **Input**: Accept a single DTO parameter for complex operations
- **Output**: Return `error` for commands, `(ResponseDTO, error)` for queries
- **No primitives**: Avoid primitive parameters; use DTOs or value objects

## Testing

- Repository interfaces are not tested directly
- Test implementations in `src/infra/{entity}/` instead
